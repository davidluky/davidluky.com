MMX Phase 4 P3 — Weapon palette probe

Loaded stage_select.mss, wrote all weapon/armor/heart/sub-tank acquired flags directly to WRAM, entered Chill Penguin via the stage-select replay, tapped SELECT 9 times, dumped 512B CGRAM after each. Goal: find X's palette row by watching which CGRAM bytes change on weapon switches.

Finding WITHDRAWN. First pass claimed row 9 is X's sprite palette because its bytes changed on some SELECT taps. David spotted the problem visually: across all 10 shots X's sprite looks identical — no weapon tint ever. The armor writes worked (you see helmet/chest/boots) but the weapon-acquired bit alone wasn't enough to make SELECT cycle weapons, so X stayed on buster the whole run. The row-9 byte changes are likely HUD weapon-icon flicker, not X. Waiting on an authentic password-entry fixture before re-running P3.
Honest caveat: my WRAM weapon-grant writes didn't fully stick. Of the 9 SELECT taps, 3 produced zero palette changes (01→02, 02→03, 05→06) — meaning SELECT bounced on the same weapon slot. MMX1 probably needs more than bit 6 at $7E1F86..$7E1F96 for a slot to be cycleable (defeated-boss flags?). Captures 01/02/03 are byte-identical and captures 04..09 include a camera scroll that triggered a zone palette bank load (large 43-byte diffs across rows 0, 1, 5, 6, 7, 8, 9). Row 9 still stands out as the weapon-correlated row, so the P3 conclusion holds.
